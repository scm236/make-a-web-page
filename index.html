Lesson 1.1 - Web basics

General notes

You (and your browser) request to the Internet, which requests to servers (closet computers hosting files). HTML stands for “HyperText Markup Language” – base for all web pages. Web pages are made up of text content (what you see) and markup (what it looks like). Web pages can contain references to other documents (images/videos/etc), links to other pages.

HTML tags (in all cases below: greater/less than signs replaced with parentheses)

Bold: (b)bold(/b)
Emphasis: (em)italicize(/em)
Image tag: (img src=”url” alt=”text”) [nice to include alt just in case url doesn’t work][“void” tag doesn’t need a closing tag]
Break tag: (br) makes a line break [another void tag], stack if you want multiple breaks
Paragraph tag: (p) content (/p) will render as a paragraph
Difference between break and paragraph tags: former is inline (just text), whereas paragraph makes a block (with height and width)
Span and div both contain content (inline and block, respectively): (span) text (/span), (div) text (/div)
Attributes: (tag ATTR=“value”)contents(/tag)
Anchors make links (a href = "http://www.reddit.com")derp(/a) links to Reddit and displays “derp”

Lesson 1.2

Basic word in HTML is a tag; browser forms tags into "tree" with DOM. CSS must be used to define style (HTML is frame of house, CSS is decoration/interior). HTML has to provide the browser information on what style system to use and where to find it. When coding different HTML elements on a page, give each different "box" its own div class so that you can keep them separate and easily apply CSS to different ones.

Lesson 1.3

Cascading style sheets (CSS)

Different levels of rules (national-state-municipal law metaphor). Most specific rule for a certain element gets applied. Inheritance allows certain CSS property values to be applied to “descendant” elements, so that properties do not have to be written over and over again.
To make a CSS rule, start with a selector (ex p { to select all paragraphs, .description { to select elements with class description)

Reverse-engineering webpages

Process for reverse-engineering towards a goal: code, test, refine, starting from general layout towards smaller details. You can use Developer Tools (inspect element in Firefox) to edit webpages, tweak them to better understand how they’re built.

Other important notes

Importance of structure

Structure is vital for computer code. First, a good structure keeps things organized, so that you can quickly and easily go through your code. This makes it easier to know which parts do what, and simplifies things like assigning CSS rules to groups and debugging. Second, because computers are "stupid" and interpret code literally with set rules, a good structure will make computers more likely to understand your code. Unstructured code may have small errors or discrepancies which will make computers unable to read your code properly.

Importance of avoiding repetition

To write an effective program, it is also important to avoid repetition. Repetition is not only inefficient and time-consuming, it can create redundancies and errors. A computer may get confused by multiple items of overlapping code in a program, impeding the code's functionality.
